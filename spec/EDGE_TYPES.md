# 模型關係邊類型規範（SSOT）

> 此檔案是 latticework 圖譜「邊」的單一真相來源。每條 related 邊都應依此規範分類並附簡短 rationale。
> 目標讀者：EMBA 學生、有經驗的企業高管。

## 6 種邊類型

| type 值 | 中文名 | 定義（一句話） | directed | 視覺 |
|---|---|---|---|---|
| `root` | 同源 | 兩個模型是同一思想的兩個切面 | 否 | 實線金 #B58B3C |
| `complementary` | 互補 | 一起用才完整，各自有死角 | 否 | 實線青 #3D6A8C |
| `antithetical` | 拮抗 | 兩種對立的世界觀或預測 | 否 | 雙線紅 #7A2330 |
| `isomorphism` | 跨域映射 | 不同學科指向同一數學/結構模式 | 否 | 虛線紫 #5C508A |
| `lollapalooza` | 互鎖 | 多個一起作用才產生極端效果（Munger 用語） | 否 | 粗實線橙 #A6553A |
| `precondition` | 前置 | A 是 B 的先決條件，有方向性 | **是** | 帶箭頭灰 #6B6660 |

## EMBA 口吻寫作守則

1. **長度**：rationale 控制在 **40–60 繁中字**，最多一句中文 + 一個短附註。
2. **直擊機制**：第一句就講清楚兩個模型「**怎麼**」連結，不講廢話、不堆術語。
3. **可帶案例**：可以用「2008 房貸危機」「Kodak」這類錨點輔助，但不必每條都用。
4. **避免**：
   - 學術術語堆疊（「在認識論層面…」「具有同構特性…」）
   - 空話（「兩者都很重要」「實務上常見」）
   - 重複定義（讀者已知這兩個模型是什麼）
5. **語氣**：類似 Andy Grove《Only the Paranoid Survive》或 Munger 演講——克制、精準、有時帶一點警語。

## 樣本（每類型至少 1 條，已通過驗證）

### root · 同源

> 機會成本 ↔ 沉沒成本（#3 ↔ #15）
> 都是金錢的方向性——機會成本盯著未來放棄的，沉沒成本盯著過去花掉的；理性決策只看前者，人性卻常困在後者。

> 損失厭惡 ↔ 稟賦效應（#33 ↔ #35）
> 都源於「失去比得到痛苦兩倍」的不對稱——稟賦效應是這條不對稱套到自有資產上的結果。

### complementary · 互補

> 能力圈 ↔ 反事實推理（#63 ↔ #61）
> 一個替你劃邊界、一個壓力測試你的判斷——前者防誤入未知，後者防自滿於已知。

> 第一原理 ↔ 奧卡姆剃刀（#58 ↔ #60）
> 一個負責拆到底、一個負責別在中間多塞——前者打破慣例，後者抗拒過度設計。

### antithetical · 拮抗

> 反射性理論 ↔ 效率市場假說（#21 ↔ #17）
> EMH 說價格反映一切資訊，反射性說價格反過來改變現實——後者解釋為何泡沫不只是「非理性」，而是市場結構本身會孕育泡沫。

> 創造性破壞 ↔ 路徑依賴（#20 ↔ #25）
> 一個說產業會新陳代謝、一個說組織會被歷史鎖定——同一現象常被兩股力量同時拉扯（Kodak 既被破壞、也是被自己路徑卡死）。

### isomorphism · 跨域映射

> 自然選擇 ↔ 創造性破壞（#49 ↔ #20）
> 演化的選擇壓力同構於市場的競爭淘汰——Schumpeter 直接借用 Darwin 的隱喻來理解產業迭代。

> 複利效應 ↔ 反饋迴路（#41 ↔ #43）
> 物理的指數成長同構於商業的飛輪——複利是時間軸上的反饋迴路。

> 臨界質量 ↔ 相變（#42 ↔ #48）
> 物理的相變同構於社群/平台的爆發點——同樣的非線性突變模式。

### lollapalooza · 互鎖

> 互惠 + 社會認同 + 權威 + 稀缺
> 四個獨立的心理槓桿同時扣下時威力相乘而非相加，這是 Munger 所稱的「Lollapalooza Effect」——Theranos 是教科書級案例。

### precondition · 前置（有方向）

> 第一原理 → 創造性破壞（#58 → #20）
> 不打破產業既有假設，就不會有真正破壞性的創新——第一原理思考是創造性破壞的認知前提。

> 規模經濟 → 網路效應（#5 → #10）
> 先有成本端規模優勢，才支撐得起用戶端網路效應的擴張投資——順序顛倒會燒光現金。

## 輸出格式

每條邊輸出一筆 JSON object：

```json
{
  "source": 3,
  "target": 15,
  "type": "root",
  "rationale": "都是金錢的方向性——機會成本盯著未來放棄的，沉沒成本盯著過去花掉的；理性決策只看前者，人性卻常困在後者。"
}
```

對於 `precondition` 類型，加 `"directed": true` 並讓 `source` 指向「先決條件方」、`target` 指向「被推導方」。其餘類型省略 `directed` 欄位（預設無向）。

## 分類決策樹（遇到難分的邊時用）

```
這兩個模型是同一條思想的不同說法？ → root
還是有方向、A 必須先發生才能引出 B？ → precondition
還是兩者世界觀對立、相互否定？ → antithetical
還是不同學科卻指向同一結構？ → isomorphism
還是必須三個以上同時作用才生效？ → lollapalooza
否則 → complementary（最常見的 fallback）
```
