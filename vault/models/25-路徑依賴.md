# #25 路徑依賴 · Path Dependence

> 學科：經濟學 ｜ 經典個案：QWERTY 鍵盤 / Microsoft

## 一句話定義
歷史的偶然選擇會被現在鎖死——即使有更好的替代方案，整個系統也難以轉換。

## 核心原理
路徑依賴指的是「過去的選擇限制了現在的可能」。一個產業、技術、或制度的當前狀態，往往不是經過完整理性比較後的最優選擇，而是早期某個偶然決定、加上後續的鎖定機制，讓系統停留在一個未必最優、但難以更換的位置。

形成路徑依賴有幾個機制。「規模經濟」：先到的標準累積大量使用者，後進者很難達到相同規模優勢。「網路效應」：每個新使用者加入既有標準會增加整體價值，讓既有標準持續強化。「學習效應」：使用者、開發者、配套產業圍繞既有標準累積知識與技能，轉換意味著大量重新學習。「協調困難」：即使所有人都同意有更好的選項，也很難協調全體同步切換——個別行動者切換早了會孤立、切換晚了會被淘汰。

路徑依賴對「最優」這個概念是一個挑戰。古典經濟學假設市場會找到最有效率的解，但路徑依賴指出：許多現存解只是「歷史的勝出者」，未必是邏輯上的最優。早期的微小優勢（或偶然事件）會被反饋機制放大，鎖定為長期均衡。

## 為什麼商業上重要
路徑依賴幫經理人理解兩件事：為什麼某些看似次佳的標準難以被取代（這是既有玩家的護城河來源），以及為什麼自己若身處新興產業，搶到「初期標準」極為重要（這決定了未來幾十年的市場結構）。投資人能用這個模型判斷哪些既有龍頭擁有路徑依賴鎖定，哪些只是暫時領先。

## 商業應用場景
- **場景一：技術標準戰**——藍光與 HD-DVD、VHS 與 Betamax、5G 規格之爭，本質都是搶建路徑依賴。
- **場景二：軟體生態系**——程式語言、作業系統、雲端平台一旦累積開發者社群，後進者極難追上。
- **場景三：地理產業聚落**——半導體在矽谷與台灣、金融在紐約倫敦、時尚在巴黎米蘭，產業聚落本身就是路徑依賴的產物。
- **場景四：組織內部流程**——公司內部的系統、流程、文化也有路徑依賴——「我們一直這樣做」可能不是最優但難以改變。

## 經典個案剖析

### 主案例：QWERTY 鍵盤 / Microsoft
QWERTY 鍵盤的故事是路徑依賴最常被引用的例子。1870 年代，打字機發明人 Christopher Sholes 設計了 QWERTY 排列，據說是為了把常用字母分散排列，減少機械打字機的字桿卡住問題。這個設計反映的是當時機械限制，並非為人類手指最佳化。

二十世紀，更人體工學的鍵盤排列陸續被提出，最著名的是 1936 年 August Dvorak 設計的 Dvorak 鍵盤。多項研究指出 Dvorak 在打字速度、疲勞度上優於 QWERTY。但近一個世紀過去，幾乎全世界仍使用 QWERTY。原因不是 QWERTY 比較好，而是路徑依賴：所有人學的是 QWERTY、所有鍵盤生產的是 QWERTY、所有打字教學針對 QWERTY、所有軟體預設 QWERTY。要轉換的話，個人切換得不到好處（沒有 Dvorak 鍵盤可用、所有公用電腦都是 QWERTY），廠商切換沒有客戶，學校教育沒誘因更新。整個系統陷在原地。

Microsoft 的故事是企業版的 QWERTY。1980 年代初期，IBM 選擇 Microsoft 的 MS-DOS 作為個人電腦作業系統，這個決定部分出於偶然（IBM 本來談的對象是另一家公司 Digital Research，後者沒談成）。一旦 IBM 標準成為主流，大量軟體開發者為 DOS（與後來的 Windows）寫程式、企業為 Windows 培訓員工、硬體廠商與 Windows 相容。

到了 1990 至 2000 年代，即使 Mac OS、Linux 在某些技術指標上優於 Windows，整個生態系仍鎖死在 Windows——個別企業切換要重寫所有軟體、重訓所有員工、處理檔案格式相容性。Microsoft 在桌面作業系統上的壟斷，與其說來自技術領先，不如說來自路徑依賴形成的生態鎖定。這是 1998 年美國司法部對 Microsoft 反壟斷訴訟的核心議題之一。

但路徑依賴不是永遠。當底層平台變遷（從桌面到行動裝置），原有的路徑依賴失效——Microsoft 在行動作業系統上完全沒有重現桌面的壟斷地位，最終由 iOS 與 Android 重新建立新的路徑依賴。這展示了路徑依賴的另一個重要特徵：它在「同一個遊戲場」內非常強大，但無法跨越遊戲場的變遷。

教訓有二。第一，既有領先者的護城河往往不是技術而是路徑依賴，要識別這層才能正確估算其價值。第二，路徑依賴對既有領先者既是防護也是陷阱——當底層平台變遷時，他們會被自己過去的優勢拖累，難以轉向。

### 延伸案例
- **電網交流電與直流電**：十九世紀末特斯拉（交流電）與愛迪生（直流電）的標準之爭，最終交流電勝出並鎖定全球電網。即使現代某些技術下直流電更有效率，整個基礎設施已不可能重來。
- **美國度量衡制度**：美國至今仍主要使用英制（英尺、磅、加侖），即使國際標準是公制。轉換成本（教科書、工程圖、產品規格）使得任何切換努力都失敗。

## 實務啟示
- 給經營者：盤點你公司內外的路徑依賴——哪些是你的護城河、哪些是讓你無法創新的枷鎖。
- 給投資人：擁有路徑依賴鎖定的公司有極強護城河，但要警覺底層平台變遷會把這層保護瞬間掀掉。
- 給創業者：在新興產業中，搶佔「初期標準」可能比產品本身更有戰略價值——標準一旦鎖定，後進者極難超越。
- 通用思維：「為什麼是這樣」的答案常常是「歷史就是這樣」——這既解釋了現實，也指出改變的方向。

## 常見陷阱
- **陷阱一：把路徑依賴鎖定的優勢視為「我們的產品比較好」**——錯把生態鎖定當技術優勢，會在平台變遷時失去防禦。
- **陷阱二：低估改變既有路徑的協調難度**——「明顯更好」的替代方案，常因協調成本而無法落地。要計算切換的總體成本，而非只看單一指標。

## 延伸思考
1. 你所在產業目前的標準（技術、流程、商業模式），有多少是「真正最優」，有多少只是「歷史鎖定」？
2. 你公司目前的核心優勢，主要來自路徑依賴鎖定還是真正的能力？這影響你該如何防禦下一波平台變遷。
